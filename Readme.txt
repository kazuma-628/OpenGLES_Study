■■■ ビルド環境生成方法 ■■■

このプログラム群は、開発環境に汎用性を持たせるために「CMake」でのビルド環境生成を採用しています。

-----------------------------------------------------------------------------------
０、「CMake」をインストールします。
　インストールされている方は「１」へ進んでください。
　
　CMakeはコンパイラに依存しないビルド自動化のためのフリーソフトウェアです。
　私が作成したVisual Studioのバージョンと、皆様の使用しているVisual Studioのバージョンが一致しないと、
　正しくコンパイルされないことが多々発生します。
　そうした問題を解消するのが「CMake」です。

-----------------------------------------------------------------------------------
１、CMakeでソリューションを生成します。
　CMakeを起動して画面上部にあるディレクトリ設定箇所に下記の設定します。
　・[Where is the source code:]にCloneした「OpenGLES_Study」フォルダへのパスを設定
　　→（例）Z:/Programming/OpenGL/OpenGLES_Study
　・[Where tp build the binaries:]にCloneした「OpenGLES_Study\Build」フォルダへのパスを設定
　　→（例）Z:/Programming/OpenGL/OpenGLES_Study/Build
　※Zipでダウンロードした場合は「OpenGLES_Study-master」となると思います。
　
　画面下部にある「Configure」を押下します。
　[Specify the generator for this profect]に「お使いのVisual Studioバージョン」を設定します
　この時「Visual Studio XX XXXX」を選ぶようにして、
　「ARM や Win64」の選択肢を選ばないようにしてください。
　また、対応しているVisual Studioのバージョンは「2010 〜 2015」です。
　※画面下部の選択肢は何もせずにそのままでOKです。
　「Finish」を押して正常終了すれば「Configuring done」が表示されます。
　
　「CMAKE_CXX_COMPILER could be fpimd.」というエラーが表示された場合は、
　C++開発ができるVisual Studioのバージョンが正しく選択されていません、
　（選択したバージョンがインストールされていないか、C++の開発環境が含まれていません）
　「メニューバー」の「File」→「Delete Chash」を押してから、本項目の最初からやり直してください。
　
　次に先ほど押下した「Configure」の隣にある「Generate」を押下します。
　正常終了すれば「Generating done」が表示されます。
　
　これで既にお使いのVisual Studioバージョンでのビルド環境（ソリューションファイル）が
　「OpenGLES_Study\Build」に生成されたはずです。

-----------------------------------------------------------------------------------
２、生成したソリューションファイルを起動します。
　「OpenGLES_Study\Build」に生成された「OpenGLES_Study.sln」を起動します。
　初めて起動するときは、ソリューションエクスプローラーで「ALL_BUILD」が太字になっていると思います。
　このままではデバック実行ができないので、
　「Main」を右クリックして「スタートアップ プロジェクトに設定」を選択してください。（初回のみでOKです）
　（面倒なので、CMakeのパラメータで自動設定できないか模索中です）

-----------------------------------------------------------------------------------
３、ビルド後デバッグ実行が可能です。

-----------------------------------------------------------------------------------
４、成果物を他人に渡す場合
　ビルドした成果物は「OpenGLES_Study\Output」に「OpenGLES_Study.exe」として生成されます。
　もし、このファイルを他人に渡すときは、
　「OpenGLES_Study\Shader」と「OpenGLES_Study\Output」をフォルダごと両方渡してください。
　このフォルダ位置関係が崩れるとプログラムが正しくシェーダーを読み込めず、エラーとなります。
　（改善したいなとは考えています）
　
※ちなみに私は「Visual Studio 2015」で動作確認をしています。
