このプログラム群は、開発環境に汎用性を持たせるために「CMake」でのソリューション生成を採用しています。

1,CMakeでソースコードのディレクトリ設定を「OpenGLES_Study」フォルダパスに設定し、
　お持ちのVisual Studioバージョンを選択して、ソリューションファイルを生成してください。
　※対応しているVisual Studioのバージョンは「2010 〜 2015」です。
　　また、Visual Studioのターゲットを選ぶ場所では、「Visual Studio XX XXXX」を選んでください。
　　（「Visual Studio XX XXXX Win64」は選ばないでください）

2,Cmakeでの作業が正常に完了すると「OpenGLES_Study.sln」というソリューションが作成されるので開いてください。
　初期設定では、ソリューションエクスプローラーで「ALL_BUILD」が太字になっていると思いますので、
　「Main」を右クリックして「スタートアッププロジェクトに設定」を押してください。
　（これしないとデバッグ実行できません、Cmakeのパラメータでなんとかできないか模索中です）

3,デバック実行が可能なはずです。

4,「DEBUG」でビルドした成果物は「Output」フォルダに格納されます。

5,ビルドした成果物の「OpenGLES_Study.exe」を移動させるときは、
　EXE単体で動かすのではなく「Output」フォルダと「Shader」フォルダごと動かしてください。
　この位置関係が崩れると、プログラム実行時にシェーダーの読み込みエラーとなります（改善したいです）
　
※作成者は「Visual Studio 2015」で動作確認をしています。
